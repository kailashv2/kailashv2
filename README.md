## Technical Approach

I do not build tutorial projects and call them portfolio pieces.
Every project I ship has three things:

**1. A real problem it solves** — not a todo app, not a weather widget.
Real use cases: document intelligence, charity transparency, coastal safety.

**2. Technical decisions I can defend** — why Drizzle ORM over Prisma,
why ChromaDB over Pinecone, why Groq LLaMA over OpenAI GPT-4.
I know what I chose and exactly why I chose it.

**3. A live deployment** — code that runs only on localhost is not a project.
My work ships.
